# Deckseer Project Status

Deckseer is currently a local, manual JSON-first decision-support coach for Slay the Spire 2. It ranks visible card reward options, always includes Skip, and explains the recommendation with deterministic run-needs scoring.

## Local MVP Handoff

Deckseer's current MVP target is a personal/local daily-use loop, not public distribution. The recommended play path is:

1. Start Slay the Spire 2 normally with the already-installed Deckseer Exporter.
2. Run `.\tools\deckseer-play.cmd` from the repo, or double-click `start-deckseer-exporter.cmd` in the repo root.
3. Pause when `deckseer export-alert` reports an important exporter state.
4. Run `deckseer inspect-export "%LOCALAPPDATA%\Deckseer\exports\latest_state.json"` first.
5. Run `deckseer recommend-export "%LOCALAPPDATA%\Deckseer\exports\latest_state.json" --confirmed --format text` only after manually confirming the visible screen matches the export.

`export-alert` plus `tools/deckseer-play.cmd` / `tools/deckseer_play.ps1` is now the current local-MVP workflow. `start-deckseer-exporter.cmd` is a repo-root click target for the same helper. The workflow is read-only/export-only: no game launch, no STS2 mod install, no input automation, no OCR/live capture, no Windows startup service, and no recommendation without explicit confirmation. Event/special routes remain `exporter_status` diagnostics only, including choose-bundle, deck-enchant, and any future choose-relic overlay proof. The alert loop stays loud for recommendation-ready exports, true choose-relic overlay proof, unexpected route shapes, post-collection mapping gaps, unsupported reward shapes, and diagnostic errors; proven deck-enchant/status-only routes, pre-collection mixed reward screens, and mapping gaps already covered by the current local repo data are quiet by default. Alert output labels whether Codex attention is useful. Public/shareable MVP packaging, OS-native notifications, and broader watch/recommend behavior remain future work.

Live-play validation for this handoff is in progress through a normal human-controlled STS2 run with the helper open. The current alert policy is: idle/cleared states do not spam; proven deck-enchant/status-only and pre-collection mixed reward states stay quiet; mapping gaps already known to the local repo catalog stay quiet even if the installed exporter mapping snapshot is older; true choose-relic overlay proof, unexpected route shapes, new post-collection mapping gaps, unsupported shapes, diagnostic errors, and recommendation-ready `card_reward` or `relic_reward` states still alert with a `Codex attention` label; stale/closed screens downgrade to `exporter_status`; and no recommendation is accepted from `exporter_status`.
